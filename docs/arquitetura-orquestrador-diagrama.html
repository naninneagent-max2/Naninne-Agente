01 Por que LangGraph 1.0 e não outra coisa

Em julho de 2026, o estado da arte consolidou: LangGraph é o padrão de produção, OpenAI Swarm foi arquivado em março de 2025, Microsoft AutoGen entrou em modo manutenção. Cinco requisitos não-negociáveis eliminam três alternativas e apontam LangGraph como única escolha séria.

5 requisitos, 1 framework

1. Persistência confiável de estado (Postgres nativo)
2. Human-in-the-loop via interrupt()
3. Paralelismo nativo entre agentes (Send API)
4. Observabilidade first-class (LangSmith)
5. Erro recovery declarativo (RetryPolicy + circuit breaker)

Posição de mercado (2026)

38% dos deployments de produção multi-agente usam LangGraph
38,8M downloads/mês no PyPI (~30× o AutoGen)
74% de task completion rate em benchmark head-to-head
41/47 falhas de tool-call recuperadas via fallback nodes

04 Cascata de modelos

Toda mensagem passa pelo Qwen3 8B (custo ~$0.0001) antes de chegar a um modelo caro. Critério: SIMPLE → Hermes 4.3 / Devstral 2 (baratos), COMPLEX → Claude Sonnet 4, contexto > 200k tokens → Gemini 2.5 Pro.

Tier 0 · Router SEMPRE RODA
Qwen3 8B classifica em simple / complex / ambiguous. Latência ~200ms. Custo ~$0.0001. Saída alimenta o PlanGen.
Tier 1 · Mecânico SIMPLE
Hermes 4.3 36B (self-hosted) — tarefas mecânicas: transcrever, etiquetar, extrair CSV, busca exata. Devstral 2 (Mistral) — código/pandas. Custo $0.001-0.10/call.
Tier 2 · Raciocínio COMPLEX
Claude Sonnet 4 — geração de plano, escrita final, decisões críticas. Latência ~2-3s, TTFB 200ms. Custo $0.10-0.50/call.
Tier 3 · Contexto gigante > 200K TOKENS
Gemini 2.5 Pro — janela de 2M tokens. Usado no caso 5.1 (lê O Príncipe inteiro + 200 páginas suas). Custo $0.30/input 2M.
Fallback EM CASCATA
Claude Sonnet 4 → timeout/5xx → Gemini 2.5 Pro → timeout → Hermes 4.3 → erro 5xx → ReportError a Robert. Circuit breaker pausa provedor com 5 falhas/60s.

07 Persistência e recuperação de falhas

PostgresSaver nativo da LangGraph 1.0 persiste snapshot do state após cada nó. Thread_id ≤ 255 chars (limitação do schema). HitL pausa o grafo com interrupt() e retoma com Command(resume=...).

Stack de persistência

  • ▸Checkpointer: AsyncPostgresSaver (tabelas checkpoints, checkpoint_blobs, checkpoint_writes)
  • ▸Long-term memory: Mem0 sobre pgvector
  • ▸File storage: Supabase Storage
  • ▸Cross-graph store: Postgres-backed Store

Recuperação de falhas

  • ▸Container reinicia: k8s/systemd reinicia; checkpointer recarrega via thread_id
  • ▸5xx de modelo: RetryPolicy (3× com backoff 1s, 2s, 4s)
  • ▸Timeout > 30s: Fallback Claude → Gemini → Hermes
  • ▸Rate limit 429: Backoff 10s, 30s, 60s, 120s, 300s
  • ▸Loop infinito: IsLastStep (25 supersteps) → pede simplificação

Exemplo de HITL (caso 5.4 — Supabase)

# Nó HumanApproval no grafo
def human_approval(state: AgentState) -> Command:
    # Pausa o grafo e serializa estado no Postgres
    decision = interrupt({
        "payload": state["plan"],
        "action_requests": [{"action": "apply_migration"}],
        "review_configs": {"decision_types": ["approve", "edit", "reject"]}
    })
    if decision["type"] == "approve":
        return Command(goto="organizador")
    elif decision["type"] == "edit":
        return Command(update={"plan": decision["changes"]}, goto="plan_gen")
    else:
        return Command(goto="__end__", update={"final_deliverable": {"cancelled": True}})

# Frontend resume
graph.invoke(Command(resume={"decisions": [{"type": "approve"}]}), config)

08 Integrações externas

Mem0 carrega contexto de Robert no início de cada thread e persiste aprendizados no fim (com aprovação). LangSmith captura traces de cada nó com custo, latência e modelo.

Mem0 — quem lê e quem escreve

▸ LÊ Memória: todo início de thread (top_k=8) ▸ LÊ Orquestrador: writing_style + projects ▸ LÊ Redator: writing_style (caso 5.1) ▸ LÊ Revisor: feedback (evita repetir erros) ▸ LÊ Bibliotecário: project mappings ▸ ESCREVE após 👍/👎 → feedback ▸ ESCREVE novo projeto detectado ▸ ESCREVE ajuste de tom aprovado

LangSmith — traces capturados

▸ run_id + thread_id + parent_run_id ▸ name (qual nó) + inputs + outputs ▸ latency_ms (P50/P95 por nó) ▸ tokens_in / tokens_out + cost_usd ▸ model_name (qual modelo) ▸ error_type + error_message ▸ human_interrupt_id (HITL) ▸ cascade_decision (auditoria da cascata)
